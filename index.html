Шпаргалка команд Git

HW #1 "VCS (Git, Github)" - Sasha Volozhanin

Base:

- git config --global user.name - имя автора данного коммита
- git config --global user.email - почта, куда будет приходить инфа о коммитах с которыми работаем
- git init - создает новый репозиторий нашего файла, запускает
- git status - проверяет статус текущей работы, что мы там наделали
- git add - добавление новых текущих изминений, файлов
- git commit - данные которые находятся в индексе, переносятся в репозиторий
- git log - история коммитов именно в даной ветки
- git checkout - выбирает данную ветку и переключается на ее, обновляя до робочего состояние
- git branch - cоздаёт новую ветку
- git push - все изминение с локального репозитория перносятся в удаленый репоз-й
- git pull - загрузка истории с удалённого репоз-я в локальный и объединяет их
- git merge - изменения ветки удалённого репоз-я в локальную текущий репоз-й
- git clone - скачивает репоз-й со всей историей
- .gitignore - чтобы файлы нашего редактора не попали в репоз-й

Advanced:

- git revert - отменяет некоторые существующие коммиты
- git reset - если увидели ошибку, можно ее отменить, с помощью этой команды
- git rebase - склеить несколько комметов в один. как я понял, если несколько человек работают на одной веткой, могут быть конфликты
- git cherry-pick - для изминение, каким-либо коммитом, и снова применить, но только заново, вверху ветки, без слияния
- git stash - при смены ветки, сохраняет наши наработки, которые мы не хотим пока вносить, просто их скрывает в хранилище, скрывает
